# Markdown-Collab 教學分組與網址派發使用手冊

這份手冊主要說明如何利用專屬網址參數 `?room=`，在教學現場快速為學生建立隔離的協作空間，並同時實現自動化的檔案歸檔管理。

## 🎯 核心概念：`?room=` 參數的功能

當我們輸入像以下這樣的網址：
`https://markdown-collab-cyan.vercel.app/?room=112/mapmo/20260406`

系統會在背後自動進行兩件重要的事情：

1. **即時編輯隔離（虛擬會議室）**：
   WebSocket 伺服器會將 `112/mapmo/20260406` 視為一個獨立的「房間名稱」。
   只有**點擊完全相同網址**的學生，才能夠互相看見對方的游標並進行即時協作。不同網址的團隊完全不會干擾彼此。

2. **自動化 GitHub 目錄歸檔（存檔路徑）**：
   當學生編輯完畢按下「儲存並發佈至文件庫」時，系統會自動辨識網址中的斜線（`/`），並對應到 GitHub 上的資料夾結構。
   您的 GitHub Repository 就會自動產生（或更新）以下檔案路徑：
   👉 `workspaces/112/mapmo/20260406.md`

---

## 👩‍🏫 給老師的實務應用建議

透過在提供給學生的網址中定義 `/`，您可以完美地幫學生進行分類。不需要事先在 GitHub 上手動建立資料夾，學生只要按下儲存，系統就會幫你蓋好資料夾結構！

### 範例一：依據「年度 / 課程 / 日期」管理（適合每日筆記或隨堂測驗）
如果您希望學生每堂課交一份共筆，可以這樣派發網址：
- `https://markdown-collab-cyan.vercel.app/?room=112/mapmo/20260406`
  👉 存入: `workspaces/112/mapmo/20260406.md`
- `https://markdown-collab-cyan.vercel.app/?room=112/mapmo/20260413`
  👉 存入: `workspaces/112/mapmo/20260413.md`

### 範例二：依據「年度 / 課程 / 組別」管理（適合分組期末報告）
若您的課程有 5 個小組正在準備期末專案，您可以這樣設計網址：
- 第一組：`...?room=112/UIUX/Group1`
  👉 存入：`workspaces/112/UIUX/Group1.md`
- 第二組：`...?room=112/UIUX/Group2`
  👉 存入：`workspaces/112/UIUX/Group2.md`

### 範例三：依據「學號」管理（適合個人作業）
如果是個人作業，您可以在 Moodle (或數位學習平台) 請學生依照學號自行改變網址：
- 學生 A：`...?room=112/Web/A1120001`
  👉 存入：`workspaces/112/Web/A1120001.md`

## 🚀 如何派發給學生？

為了避免學生打錯網址跑到錯誤的房間去，推薦以下幾種方式來派發給他們：

1. **數位學習平台（如 Zuvio、Moodle）**：上課前，老師先準備好各組的連結，直接貼在公告區讓學生點擊。
2. **QR Code 產生器**：各別將每一組的專屬網址做成 QR Code 放上簡報，學生拿手機/平板掃描直接進入。
3. **網址縮短服務（如 bit.ly 或 reurl）**：將超長網址變短，學生方便在電腦上自行輸入。例如將 `...?room=112/mapmo/group1` 縮短為 `reurl.cc/mapmo-g1`。

## ⚠️ 注意事項與常見提醒

- **路徑中不能有特殊字元與空格**：請叮嚀學生，若需自訂名稱，只使用「英文、數字、減號 `-` 與底線 `_`」，避免遇到網址解析異常。
- **認明相同的 URL**：如果學生抱怨「為什麼沒看到組員？」，請務必要求他們檢查網址列最後的 `room=...` 是否完全一致（大小寫也必須一模一樣）。
